## MDX

MDX is a powerful way to write content with components. It's a combination of
Markdown and React.

## Available Components

shadcn/ui provides a wide range of components that you can use directly in your MDX files. Here are some commonly used components:

### UI Components

- `Button` - Interactive button component
- `Input` - Form input fields
- `Alert` - Displays important messages or notifications
- `Table` - Structured data display
- `Skeleton` - Loading state placeholders
- `Form` components - For building interactive forms
- `Accordion` - Collapsible content sections
- `Toast` - Temporary notifications
- `Combobox` - Advanced selection inputs
- `NavigationMenu` - Website navigation components
- `Breadcrumb` - Path navigation components
- `Carousel` - Slideshow components
- `DatePicker` - Date selection components
- `InputOTP` - One-time password inputs
- `Resizable` - Resizable panel components

### Documentation Components

- `ComponentPreview` - Displays live component previews
- `ComponentSource` - Shows component source code
- `Steps` - Creates step-by-step guides
- `Callout` - Highlights important information
- `CodeTabs` - Organizes code examples in tabs
- `TabsList` and `TabsTrigger` - Creates tabbed interfaces
- `DataTable` - Advanced table with sorting and filtering

## Using Components

To use components in your MDX files:

1. Import the components at the top of your MDX file (if required by your setup)
2. Use them directly in your MDX content

```mdx
import { Button } from "@/components/ui/button"
import { Alert, AlertDescription } from "@/components/ui/alert"

# My Page

<Button>Click me</Button>

<Alert>
  <AlertDescription>
    This is an important message.
  </AlertDescription>
</Alert>
```

## Best Practices

1. **Component Organization**
   - Group related components together
   - Use semantic structure (headings, sections)
   - Keep content well-organized with appropriate spacing

2. **Accessibility**
   - Include proper ARIA labels
   - Maintain heading hierarchy
   - Provide alt text for images
   - Use semantic HTML elements
   - Ensure keyboard navigation works

3. **Performance**
   - Import only the components you need
   - Use `ComponentPreview` for complex examples
   - Leverage code splitting when possible
   - Optimize images and media content
   - Use loading states appropriately

4. **Styling**
   - Use consistent spacing and layout
   - Follow the shadcn/ui design system
   - Apply custom styles through className props
   - Maintain responsive design principles
   - Use theme variables for consistency

5. **Documentation**
   - Include examples for complex components
   - Document props and variants
   - Provide context for usage
   - Show common patterns and combinations
   - Include error handling examples

## Advanced Usage

### Composing Components

You can compose multiple components together to create more complex interfaces:

```mdx
<Card>
  <CardHeader>
    <CardTitle>User Profile</CardTitle>
  </CardHeader>
  <CardContent>
    <Form>
      <Input placeholder="Username" />
      <Button>Save</Button>
    </Form>
  </CardContent>
</Card>
```

### Interactive Examples

Use `ComponentPreview` to show interactive examples:

```mdx
<ComponentPreview
  name="button-demo"
  description="A collection of button variants."
/>
```

### Data Tables

Create advanced data tables with sorting and filtering:

```mdx
<DataTable
  columns={columns}
  data={data}
  searchKey="title"
  pagination
  filtering
/>
```

### Responsive Layouts

Use resizable components for flexible layouts:

```mdx
<ResizablePanelGroup direction="horizontal">
  <ResizablePanel>Sidebar</ResizablePanel>
  <ResizableHandle />
  <ResizablePanel>Main Content</ResizablePanel>
</ResizablePanelGroup>
```

### Loading States

Implement loading states with skeleton components:

```mdx
<Skeleton className="h-[20px] w-[100px] rounded-full" />
<Skeleton className="h-[200px] w-[100%] rounded-md" />
```

Remember that all shadcn/ui components are built with accessibility in mind and follow best practices for web development. Use them consistently throughout your documentation for the best user experience.
